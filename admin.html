Why Supabase? — Solving the Storage Problem

The free tier handles approximately 50,000 asset records with photos. Photos are stored as URLs (not embedded base64), which is why you can now register unlimited assets — the database stores a link to the photo, not the photo itself.

Step 1 — Create Supabase Account

Go to supabase.com → Sign Up → Create New Project → choose a region close to your users (e.g. West Africa → use EU West). Note your Project URL and anon/public API key from Settings → API.

Step 2 — Run This SQL in Supabase SQL Editor

Open your Supabase project → SQL Editor → New Query → paste and run:

-- AssetTrack Pro — Multi-Company Database Schema -- Run this entire block in Supabase SQL Editor -- 1. COMPANIES TABLE create table if not exists companies ( id text primary key, name text not null, industry text default 'General', parent_id text references companies(id), reg_number text, contact_person text, contact_email text, contact_phone text, address text, currency text default 'NGN', fy_end text default 'December', active boolean default true, created_at timestamptz default now() ); -- 2. ASSETS TABLE create table if not exists assets ( id text primary key, barcode text not null, company_id text references companies(id), name text not null, category text, sub_category text, location text, department text, status text default 'Active', condition text default 'Good', value numeric default 0, custodian text, serial text, purchase_date date, notes text, photos text[], registered_by text, registered_from_audit boolean default false, last_audit_date timestamptz, last_audit_by text, last_audit_changes text, created_at timestamptz default now(), updated_at timestamptz default now() ); create unique index if not exists assets_barcode_company on assets(barcode, company_id); -- 3. USERS TABLE create table if not exists users ( id uuid primary key default gen_random_uuid(), username text unique not null, password_hash text not null, name text, role text default 'field', company_id text references companies(id), active boolean default true, created_at timestamptz default now() ); -- 4. CONFIG TABLE (taxonomy, locations, depts — scoped per company) create table if not exists config ( id uuid primary key default gen_random_uuid(), company_id text references companies(id), key text not null, value text not null, updated_at timestamptz default now(), unique(company_id, key) ); -- 5. AUDIT LOG TABLE create table if not exists audit_log ( id uuid primary key default gen_random_uuid(), asset_id text references assets(id), company_id text references companies(id), barcode text, action text, changes text, audited_by text, audited_at timestamptz default now(), notes text ); -- 6. ROW LEVEL SECURITY alter table companies enable row level security; alter table assets enable row level security; alter table users enable row level security; alter table config enable row level security; alter table audit_log enable row level security; create policy "Read companies" on companies for select using (true); create policy "Insert assets" on assets for insert with check (true); create policy "Read assets" on assets for select using (true); create policy "Update assets" on assets for update using (true); create policy "Delete assets" on assets for delete using (true); create policy "Read users" on users for select using (true); create policy "Insert users" on users for insert with check (true); create policy "Update users" on users for update using (true); create policy "Delete users" on users for delete using (true); create policy "Read config" on config for select using (true); create policy "Write config" on config for insert with check (true); create policy "Update config" on config for update using (true); create policy "Read audit_log" on audit_log for select using (true); create policy "Insert audit_log" on audit_log for insert with check (true); -- 7. STORAGE BUCKET for photos insert into storage.buckets (id, name, public) values ('asset-photos', 'asset-photos', true) on conflict do nothing; create policy "Upload photos" on storage.objects for insert with check (bucket_id = 'asset-photos'); create policy "View photos" on storage.objects for select using (bucket_id = 'asset-photos'); -- 8. SEED DEFAULT COMPANIES insert into companies (id, name, industry) values ('co-main', 'Main Company', 'General') on conflict do nothing; -- 9. SEED DEFAULT USERS insert into users (username, password_hash, name, role, company_id) values ('admin', 'admin123', 'System Administrator', 'admin', 'co-main') on conflict do nothing; insert into users (username, password_hash, name, role, company_id) values ('field.officer', 'field123', 'Field Officer', 'field', 'co-main') on conflict do nothing;
Step 3 — Update Your Credentials in Both Files

Open both index.html (field app) and admin.html in a text editor and replace the placeholder lines near the top of the <script> section:

// Replace these two lines in BOTH files: const SUPABASE_URL = 'YOUR_SUPABASE_URL'; const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY'; // With your actual Supabase values from Settings → API, e.g.: const SUPABASE_URL = 'https://abcdefghijkl.supabase.co'; const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

Then push both files to GitHub. Your data saves to the cloud permanently and all field officers' registrations appear in the admin dashboard in real time.
